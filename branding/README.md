# Vyrnix Linux – Branding & Design Guidelines

## Overview

This document defines the visual and conceptual design principles of the Vyrnix Linux project.

Vyrnix Linux is a modular, Arch-based Linux distribution inspired by FreeBSD design principles.
The branding reflects the system’s core philosophy: **minimal, modular, technical, and precise**.

---

## Core Identity

**Keywords**

* modular
* minimal
* technical
* structured
* performant
* controllable
* BSD-inspired
* Arch-based

**Core Message**

> A modular Linux system with a minimal base, hardware-aware configuration, and clear system structure.

---

## Design Principles

### 1. Minimal over decorative

No unnecessary visual elements. Every component must have a clear purpose.

### 2. Structure over aesthetics

Clarity and readability are more important than visual effects.

### 3. Modularity must be visible

The design should reflect system architecture:

* base system
* kernel
* modules
* hardware detection
* system roles

### 4. Technical authenticity

Vyrnix should look like a system tool, not a marketing product.

### 5. Performance as a visual concept

The design should feel:

* fast
* responsive
* lightweight
* precise

* JetBrains Mono
* Alternative: IBM Plex Mono

* no decorative fonts
* readable line length

---

## UI Guidelines

### General

* dark theme first
* high contrast
* minimal visual noise
* consistent spacing

### Components

**Buttons**

* rectangular or slightly rounded
* no excessive effects
* red used sparingly for primary actions

**Cards / Panels**

* subtle borders
* no heavy shadows
* clean separation

**Navigation**

* left-aligned
* predictable structure
* no experimental layouts

**Code Blocks**

* clearly separated
* optimized for readability
* copy-friendly

## Design Philosophy Summary

Vyrnix Linux is not designed to impress visually.

It is designed to be:

* understandable
* predictable
* efficient
* modular

The design must always support the system’s purpose — never distract from it.
